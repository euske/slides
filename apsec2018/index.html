2. Why Code Comments?

  • Good:
    • Make up the lack of documentation. 👍
    • Flexible and multi-purpose. 👍
  • Bad:
    • Unstructured. 👎
    • Hard for formal analysis. 👎

4. Research Questions

  1. What are the general structure of local comments?
  2. How can we use them for program comprehension?

Methodology

  1. Manually review comments and discover the commonality.
  2. Build a classifier to test the hypothesis.

5. Key Observation

Code comment forms a relationship.

6. Three Elements of Comment

  • Comment Extent
  • Comment Category
  • Comment Target

7. Comment Extent

  • A single meaningful sentence is sometimes split into multiple comment tags: 
    1. /* This is
one comment extent. */
    2. // This is still
// one comment extent.

8. Comment Category

  • Type of role that each comment play:
    1. Explains what the code does: (Postcondition) 
      // create some test data
Map<String, String> data = createTestData(testSize);
    2. Explains why the code is there: (Precondition) 
      // Unable to find the specified document.
return Status.ERROR;
    3. Value Description: 
      // onActivityResult request
static final int REQUEST_CODE = 6384;

9. Comment Target

  • Syntax element that a comment refers to.
    • Parent: Parent syntax element.
    • Left: Previous sibling.
    • Right: Next sibling.

10. Key Observation (Again)

  • Comment = relationship.
  • Each comment has three elements.

Annotating Comment Extents

  • IOB (BIO) notation:
    • B: Beginning of Comment
    • I: Inside Comment
    • O: Other

Annotating Comment Categories

  • We reviewed 1,000 comments and defined 11 categories.
  • Three people manually categorized 100 comments.
    • Fleiss' Kappa = 0.49 (moderate agreement)

14. Findings

  • Reasonable accuracy for the two biggest categories (Postcondition and Precondition).
  • Some characteristics (Postcondition) are preserved across different languages (Java and Python).
  • This suggests the existence of the "Universal Grammar" of code comments among programmers.
