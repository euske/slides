Yusuke Shinyama, Jan. 2023

1. ソフトウェア開発において重要なこと (復習)

開発者はみな、以下のようなソフトウェアを開発したい:

  • 高速で、効率的で
  • 安全で、信頼性があって
  • 保守しやすく、拡張性にも優れている
こんなのは嫌だ
こんなのを作りたい

しかしこれはとてつもなく難しい。 なぜなら、ソフトウェアというものは非常に複雑だからである。 この複雑さは、そもそもソフトウェアが扱っている問題（社会）の複雑さに起因する。 過去数十年、ソフトウェアの先人達はこの複雑なシステムを分解 (separation of concerns) し、 人間の小さい脳が扱えるようにするために、さまざまな技法をあみだしてきた。 プログラミング技術の歴史とは、ほとんど「分解する技術の歴史」といってもいい。

  • 1970年代: 構造化（モジュール化）
  • 1980年代: OOP (カプセル化、データ抽象)
  • 1990年代: AOP (複数箇所をまとめる)
  • 1980年代: 関数型 (状態の除去)
  • 2000年代: フレームワーク, TDD (IoC, DI, etc.)
  • ...

2. Springを使うと何ができるのか

結論から言ってしまうと、Spring Framework を使うと 以下のようなコードが: 

class Client {
    Client() {
        this.connection = new Connection("www")
    }
    get() {
        return this.connection.fetch()
    }
}

class Database {
    store(data) { ... }
}

class App {
    main() {
        client = new Client()
        db = new Database()
        data = client.get()
        db.store(data)
    }
}

以下のように書ける:

class Client {
    Client(Connection connection) { 
        this.connection = connection
    }
    get() {
        return this.connection.fetch()
    }
}

class Database {
    store(data) { ... }
}

class App {
    main(Client client, Database db) {
        data = client.get()
        db.store(data)
    }
}

ソフトウェアにおける「フレームワーク」とは、 処理の流れがプログラム本体に書かれておらず、外から制御されるような 構造のことである。 (Inversion of Control)

  • 各制御フロー、データフローが「ぶった切られている」
  • Client, Database, Connection など直接作成されていない
  • Client, Database, Connection などに別のクラスが使える
