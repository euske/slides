やったこと

  • 実行された命令の完全なトレースを効率よく保存する。
    • 0, 1, 2, 3, 4, 7, 945, 946, 950, 952, 8, 9, ...
  • 従来手法の問題:
    • 完全な記録をとるとデータ量が膨大:
      4バイト × 1G命令/s × 2スレッド = 8GB/s
      (参考: フルHD動画 60fps 無圧縮 = 373MB/s,
         SATA 理論値: 2GB/s, USB3 理論値: 625MB/s)
    • 数値の羅列として圧縮すると、解析がめんどう。

基本的なアイデア

ほとんどのプログラムでは同じ部分が何度も実行される。

基本的なアイデア

共通する部分を DAG としてまとめる。

階層構造を反映 = "Hierarchical Program Paths"

処理の流れ

  1. プログラムを、呼び出しを含まない 直列な部分 (BL path) に分割し、実行時にそのIDを記録する。
  2. 呼び出しを子ノードとして記録する (HPPTree)。
  3. 木構造をまとめて DAG にする (HPPDAG)。
    [2. と 3. は同時に実行可能。]

BL Path の例

"1,2,3,4,1,2,3,5,
  2,6,7,5,2,6,7,9,8,…"

id2"2,6,7,8"
id3"1,2,3,5"
id4"2,6,8"
id5"9"
......
